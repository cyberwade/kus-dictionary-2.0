# Техническое задание (шаблон)

> **Как пользоваться:** это ТЗ составлено по готовому проекту Kus Dictionary 2.0. Для нового проекта скопируйте файл, замените название проекта и описание, затем заполните или скорректируйте разделы по аналогии. Разделы с пометкой «шаблон» — типовые формулировки, их можно оставить или адаптировать.

---

## 1. Общие сведения

| Поле | Значение |
|------|----------|
| **Название проекта** | Kus Dictionary 2.0 |
| **Назначение** | Веб-приложение для тренировки испанских выражений и историй: при входе пользователь выбирает модуль **«Выражения»** или **«Банк историй»**. В модуле «Выражения» — разделы → ключевые фразы → предложения (таблица sentences); в модуле «Банк историй» — те же разделы → истории → предложения (таблица historia_sentences). Добавление, редактирование и удаление предложений; генерация примеров через OpenAI (опционально). |
| **Целевая платформа** | Браузер (десктоп, адаптивная вёрстка). |

---

## 2. Цели и границы проекта

**В рамках проекта:**
- Выбор модуля при входе: «Выражения» или «Банк историй».
- Модуль «Выражения»: разделы → ключевые фразы (keyphrases) → предложения (таблица sentences); пагинация, перемешивание, скрытие карточек; добавление, редактирование и удаление карточек (keyphrases); CRUD предложений.
- Модуль «Банк историй»: те же разделы → истории (historias) → предложения (таблица historia_sentences); тот же набор фильтров и карточек; добавление, редактирование и удаление карточек (historias); CRUD предложений историй.
- CRUD карточек (keyphrases, historias) в обоих модулях через UI: кнопка «Добавить карточку» в ряду пагинации, иконки редактирования и удаления на каждой карточке; при удалении карточки удаляются и все её предложения (интерфейс предупреждает).
- CRUD предложений в обоих модулях (отдельные таблицы: sentences для выражений, historia_sentences для историй).
- Генерация предложения-примера через OpenAI (опционально; в модалке добавления используется контекст фразы/истории).
- Хранение данных в PostgreSQL.

**Вне рамок (не реализуется):**
- Переименование и удаление разделов (chapters) через интерфейс — через БД при необходимости. Создание нового раздела реализовано в UI: кнопка «Добавить раздел» при выборе «Выберите раздел», ввод названия, POST /api/chapters.
- Авторизация и разграничение прав.
- Мобильное нативное приложение.

## 4. Функциональные требования

### 4.1 Выбор модуля и разделы

| ID | Требование | Реализация |
|----|------------|------------|
| F-0.1 | При входе на сайт пользователь выбирает модуль | Экран выбора «Выражения» / «Банк историй» (ModuleChooser); после выбора отображается соответствующий модуль; кнопка «Сменить модуль» возвращает к выбору. |
| F-0.2 | Общие разделы для обоих модулей | Один источник: `GET /api/chapters`; разделы те же в модуле Выражения и в модуле Банк историй. |

### 4.2 Разделы (chapters)

| ID | Требование | Реализация |
|----|------------|------------|
| F-1.1 | Отображение списка разделов для выбора | Выпадающий список в шапке; данные с `GET /api/chapters`. |
| F-1.2 | Добавление нового раздела через интерфейс | При выборе «Выберите раздел» справа от фильтра — кнопка «Добавить раздел» (оформление как кнопка добавления карточки); модалка с полем «Название раздела»; POST /api/chapters; после успеха обновление списка разделов и выбор созданного раздела. |
| F-1.3 | В модуле «Выражения» при выборе раздела загружать ключевые фразы | Запрос `GET /api/keyphrases?chapter_id=...`. |
| F-1.4 | В модуле «Банк историй» при выборе раздела загружать истории | Запрос `GET /api/historias?chapter_id=...`. |

### 4.3 Ключевые фразы (keyphrases, модуль Выражения)

| ID | Требование | Реализация |
|----|------------|------------|
| F-2.1 | Отображение списка выражений выбранного раздела | Карточки (компонент Card); пагинация (20 / 50 / все). |
| F-2.2 | Пагинация: выбор размера страницы и номера страницы | Селект «Карточек на странице», кнопки страниц. |
| F-2.3 | Перемешивание порядка карточек | Кнопка «Перемешать» — случайный порядок без повторного запроса к API. |
| F-2.4 | Возможность скрыть блок предложений по карточкам | Переключатель «Предложения: Показывать». |
| F-2.5 | На карточке: номер по порядку, комментарий к фразе, фраза (рус/исп) с раскрытием перевода | Компонент Card; кнопка «Показать» для отображения испанского варианта. |
| F-2.6 | Сокрытие карточки (ключевой фразы) из списка тренировки | Переключатель «Скрыть» на карточке; PATCH `/api/keyphrases/:id` с `is_hidden`; после сохранения карточка убирается из списка локально (без перезагрузки страницы и без сброса перемешивания/нумерации). |
| F-2.7 | Просмотр только скрытых или только нескрытых карточек | Переключатель «Скрытые карточки»; рядом с подписью — количество скрытых карточек раздела, например «Только скрытые (34)». Счётчик обновляется сразу при сокрытии/отмене сокрытия карточки (локальный state, без перезапроса); при загрузке списка (смена раздела, страницы, переключение фильтра) значение берётся из ответа GET (`total_hidden`). При переключении фильтра — перезагрузка списка с сервера (GET с `show_hidden=only` или `exclude`), сброс перемешивания и локального состояния. |
| F-2.8 | Добавление карточки выражения в раздел | Кнопка с иконкой «+» справа в ряду пагинации; модалка с полями keyphrase_esp, keyphrase_rus, keyphrase_comment (опционально); POST /api/keyphrases; после успеха — сообщение «Карточка добавлена.» внизу модалки. |
| F-2.9 | Редактирование карточки выражения | Иконка редактирования на карточке; модалка с полями esp, rus, comment; валидация как у предложений (esp и rus обязательны); PATCH /api/keyphrases/:id. |
| F-2.10 | Удаление карточки выражения | Иконка удаления на карточке; модалка с предупреждением «Вместе с карточкой будут удалены все предложения»; DELETE /api/keyphrases/:id; после удаления — перезагрузка списка. |

*Аналогичные требования F-2.x выполняются в модуле «Банк историй» для историй (historias): те же фильтры, пагинация, перемешивание, скрытие карточек; добавление (POST /api/historias), редактирование (PATCH), удаление (DELETE) карточек; данные с `GET /api/historias`.*

| F-3.2 | Добавление предложения | Модальное окно: поле additional_comment с иконками быстрой вставки («Это должен быть вопрос.»; «Используй Imperativo Affirmativo.»; «Используй Imperativo Negativo.»; очистка поля); опционально генерация через OpenAI, затем ввод/редактирование текста и сохранение (POST /api/sentences). |

## 7. Модель данных (БД)

| Таблица | Назначение |
|---------|------------|
| **chapters** | Разделы (общие для обоих модулей). |
| **keyphrases** | Ключевые выражения (модуль «Выражения»), привязанные к разделу. |
| **sentences** | Предложения-примеры к ключевой фразе (модуль «Выражения»). |

| keyphrase_rus | TEXT | | Выражение на русском. |

| Поле | Тип | Ограничения | Описание |

| Поле | Тип | Ограничения | Описание |

## 8. API (контракт backend)

Базовый префикс: `/api`.

### 8.1 Разделы

| Метод | URL | Тело/Параметры | Ответ | Коды ошибок |
|-------|-----|----------------|-------|-------------|
| GET | /chapters | — | `[{ chapter_id, chapter_name }]` | 5xx при ошибке БД |
| POST | /chapters | Body: `{ chapter_name: string }` | `{ chapter_id, chapter_name }` | 400 (пустое название), 5xx |

### 8.2 Ключевые фразы

| Метод | URL | Тело/Параметры | Ответ | Коды ошибок |
|-------|-----|----------------|-------|-------------|
| GET | /keyphrases | Query: chapter_id (обяз.), page=1, per_page=20 (0 = все), show_hidden=only \| exclude (опц.) | `{ items, total, total_hidden, page, per_page, total_pages }`; total_hidden — число скрытых карточек раздела; элемент: keyphrase_id, chapter_id, keyfrase_comment, keyphrase_esp, keyphrase_rus, order_in_chapter, is_hidden | 5xx |
| POST | /keyphrases | Body: chapter_id, keyphrase_esp, keyphrase_rus, keyfrase_comment (опц.) | `{ keyphrase_id }` | 5xx |
| PATCH | /keyphrases/{keyphrase_id} | Body: опционально is_hidden, keyphrase_esp, keyphrase_rus, keyfrase_comment | Обновление карточки | 404, 5xx |
| DELETE | /keyphrases/{keyphrase_id} | — | Удаление карточки и всех её предложений | 404, 5xx |

### 8.3 Предложения (модуль Выражения, таблица sentences)

| Метод | URL | Тело/Параметры | Ответ | Коды ошибок |
|-------|-----|----------------|-------|-------------|

| POST | /sentences/generate | Body: keyphrase_id, keyphrase_esp, keyphrase_rus, keyfrase_comment, additional_comment | `{ sentence_esp, sentence_rus }`; запрос пишется в openai_requests.log | 502, 503, 500 |
| POST | /sentences | Body: keyphrase_id, sentence_rus, sentence_esp | `{ ok: true }` | 201, 5xx |
| PATCH | /sentences/{sentence_id} | Body: sentence_rus, sentence_esp | `{ ok: true }` | 404, 5xx |
| DELETE | /sentences/{sentence_id} | — | `{ ok: true }` | 404, 5xx |

### 8.4 Истории (модуль Банк историй)

| Метод | URL | Тело/Параметры | Ответ | Коды ошибок |
|-------|-----|----------------|-------|-------------|

## 9. Структура приложения (архитектура)

### 9.1 Backend

- **Точка входа:** `app.main:app` (FastAPI).
- **Конфигурация:** `app.config` — чтение .env, объект settings.
- **БД:** слой доступа без ORM: `app.db.connection` (менеджер подключения), `app.db.repositories` (функции запросов).
- **API:** роутеры в `app.api.routes` (chapters, keyphrases, sentences, historias, historia_sentences); подключение в main с префиксом /api.
- **Внешние интеграции:** генерация текста через OpenAI по шаблону из файла (например, sentence_prompt.txt).
- **Логирование OpenAI:** модуль `app.openai_request_log` записывает каждый запрос к OpenAI (POST /sentences/generate и POST /historia-sentences/generate) в файл `backend/openai_requests.log`: время, источник, входные параметры, промпт, ответ API.

### 9.2 Frontend

- **SPA:** одна точка входа (index.html → main.jsx → App.jsx). App отображает выбор модуля (ModuleChooser) или один из модулей (ExpressionsModule, HistoriasModule).
- **Запросы к API:** модуль api/index.js — функции для chapters (getChapters, createChapter), keyphrases, sentences (Выражения), historias и historia-sentences (Банк историй).
- **Состояние:** локальный state React; без глобального стора.
- **UI:** переиспользуемая карточка (Card) для выражений и историй: переключатель «Скрыть» (16px), иконки «Редактировать карточку» и «Удалить карточку», блок предложений; модалки AddSentenceModal, EditSentenceModal, AddCardModal, EditCardModal, AddChapterModal. При выборе «Выберите раздел» справа от фильтра — кнопка «Добавить раздел» (оформление как кнопка добавления карточки); AddChapterModal: поле «Название раздела», вызов createChapter (POST /chapters), после успеха — обновление списка разделов и выбор созданного. В AddSentenceModal у поля additional_comment — иконки в кружке: вставка «Это должен быть вопрос.» (?), «Используй Imperativo Affirmativo.» (+), «Используй Imperativo Negativo.» (−), справа — очистка поля (✕ в кружке). Кнопка «Добавить карточку» (иконка «+») в ряду пагинации; при удалении карточки — предупреждение об удалении предложений. Иконки и переключатель на карточке выровнены по одной линии (24px) с иконками у предложений.

### 9.3 Конфигурация окружения

- **Backend:** DB_HOST, DB_PORT, DB_NAME, DB_USER, DB_PASSWORD; OPENAI_API_KEY, OPENAI_MODEL.
- **Frontend (при необходимости):** VITE_API_BASE для указания базового URL API.

## 10. Критерии приёмки (чек-лист)

- [ ] Запуск backend и frontend по инструкции из README; выполнены миграции add_historias.sql и add_historia_sentences.sql при использовании Банка историй.
- [ ] При входе отображается выбор модуля «Выражения» / «Банк историй»; после выбора открывается соответствующий модуль; кнопка «Сменить модуль» возвращает к выбору.
- [ ] При выборе «Выберите раздел» в обоих модулях справа от фильтра отображается кнопка «Добавить раздел»; по нажатию открывается модалка с полем «Название раздела»; после сохранения раздел создаётся (POST /api/chapters), список разделов обновляется и новый раздел выбирается.
- [ ] Модуль «Выражения»: выбор раздела → ключевые фразы с пагинацией и опцией «все»; перемешивание; переключатели предложений и скрытых карточек; кнопка «+» в пагинации — добавление карточки; на карточке — иконки редактирования и удаления; скрытие карточки убирает её из списка без перезагрузки; CRUD предложений (таблица sentences); генерация (при OpenAI).
- [ ] Модуль «Банк историй»: те же разделы; выбор раздела → истории с теми же фильтрами и пагинацией; кнопка «+» в пагинации — добавление карточки истории; на карточке — иконки редактирования и удаления; карточки историй с полями historia_comment, historia_esp, historia_rus; предложения загружаются из historia_sentences; добавление/редактирование/удаление карточек и предложений историй работает.
- [ ] При отсутствии OPENAI_API_KEY генерация возвращает понятную ошибку (503 или сообщение).
- [ ] Каждый запрос к OpenAI (sentences/generate и historia-sentences/generate) записывается в backend/openai_requests.log (время, параметры, промпт, ответ).
- [ ] В модалке «Добавить предложение» у поля additional_comment отображаются иконки быстрой вставки (вопрос, Imperativo Affirmativo, Imperativo Negativo) и очистки поля (✕).
- [ ] Ошибки API отображаются пользователю в интерфейсе.
- [ ] README и/или ТЗ содержат описание структуры, API и требований к БД (включая historias и historia_sentences).
