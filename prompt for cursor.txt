Создаем программу для тренировки иностранных выражений.
Данные хранятся в БД PostgresSQL в следующих таблицах:

chapters (
    chapter_id int4 PRIMARY KEY,
    chapter_name TEXT
);

public.keyfrases (
    chapter_id int4,
    keyfrase_id BIGSERIAL PRIMARY KEY,    
    keyfrase_comment TEXT,
    key_frase_esp TEXT,
    key_frase_rus TEXT
);

public.sentences (
    keyfrase_id int4, 
    sentence_id BIGSERIAL PRIMARY KEY, 
    sentence_esp TEXT,
    sentence_rus TEXT
);

Функциональные требования:

Описание интерфейса:
1 блок - Выбор раздела (chapter_name)
2 блок - Выбор количества карточек на странице(внутри выбранного раздела) - выпадающий список - 20, 50, Все 
3 блок - Пагинация (список страниц) в зависимости от выбора в блоке 2. Здесь же кнопка "Перемешать" - перемешивает только карточки, находящиеся на текущей стринице.
4 блок - Карточки выражений (внутри выбранного раздела). Каждая карточка содержит:
- Порядковый номер  карточки на странице (Это НЕ id выражения). При перемешивании этот номер не меняется. Здесь доступна кнопка "Добавить предложение" - - пока без функционала - по нажатию выводит модальное окно с текстом "Функция добавления пока недоступна".
- Комментарий к выражению (keyfrase_comment)
- блок выражения - Выражение на русском (keyfrase_rus) + Интерактивную ссылку с текстом "Показать". При нажатии на ссылку текст превращается в keyfrase_esp.
- блок предложений - аналогично блоку выражения, но одному выражению может  соответствовать несколько предложений (sentence_rus - sentence_esp). Для каждого предложения доступна кнопка "Удалить" - пока без функционала - по нажатию выводит модальное окно с текстом "Функция удаления пока недоступна".

Прочие:
При перемешивании карточек страница не перезагружается. 
При переходе по пагинированным страницам, перемешивание карточек слетает.
При смене количества карточек на странице, страница перезагружается, перемешивание карточек слетает, пользователю отображается первая страница.
При добавлении\удалении\редактировании предложения в карточку(в будущем), страница не перезаружается, только обновляется список предложений в карточке. 

Нефункциональные требования:
Фронтенд реализован на React.
Бэкэнд реализован на Python.
Реализованы вызовы api методов.
Данные хранятся в БД PostgresSQL.
Должна быть модульная структура приложения.
Должна быть возможность масштабирования и добавления нового функционала.
Данные для подключения к БД и OpenAI api находятся в файле проекта .env.
Должна быть верстка для экрана ПК,телефона, планшета.
Интерфейс простой не перегруженый.

Нужно реализовать новый функционал карточки: «Добавить предложение».

Общее описание логики:
При нажатии на иконку «Добавить предложение» открывается модальное диалоговое окно.

1. Интерфейс модального окна - Входной блок:
— Отображаются в виде label следующие параметры:
  key_frase_rus
  key_frase_esp
  keyfrase_comment
— Отображается поле ввода (textarea или input) для additional_comment.

Справа расположена кнопка «Создать предложение».

2. Логика кнопки «Создать предложение»

При нажатии:
— Вызывается backend API метод генерации предложения.
— Метод принимает параметры:
  keyfrase_id
  key_frase_esp
  key_frase_rus
  keyfrase_comment
  additional_comment
— Метод подставляет параметры в промпт, который хранится в файле проекта:
  /sentence_prompt.txt
— Затем метод обращается к OpenAI API.
— Метод возвращает:
  sentence_esp
  sentence_rus

3. Интерфейс модального окна - Выходной блок:

После получения ответа от API:
— Заполнить поля ввода:
  sentence_esp
  sentence_rus
— Эти поля доступны для ручного редактирования пользователем.

4. Кнопка «Добавить предложение»

При нажатии:
— В таблицу sentences базы данных добавить запись со следующими полями:
  keyfrase_id
  sentence_rus
  sentence_esp
— После успешного сохранения:
  окно НЕ закрывается
  пользователь может снова нажать «Создать предложение» и добавить ещё одно

5. Закрытие окна

— Окно закрывается по нажатию на крестик в правом верхнем углу.
— При закрытии:
  все поля формы очищаются
  additional_comment очищается
  sentence_esp и sentence_rus очищаются

6. Обновление UI

После добавления предложения:
— Карточка выражения должна обновиться динамически.
— В списке предложений должны появиться новые записи.
— Полная перезагрузка страницы НЕ допускается.
— Использовать обновление состояния (state) или повторный запрос списка предложений.

Реализовать:
— UI компонент модального окна
— Обработчики событий
— Вызов API
— Обновление состояния без reload страницы
— Обработку ошибок API (показ уведомления)
— Индикатор загрузки во время генерации предложения
